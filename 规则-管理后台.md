# 规则 — 管理后台（业务）

> 产品：luckdate · 版本 V2.0 · 2026-07-24  
> 范围：运营 / 客服在后台要完成的业务能力与操作流程。不含技术实现。  
> 对应主文档：`PRD-luckdate.md`

---

## 1. 后台要解决什么

支撑 App 业务闭环可运营：

1. 查用户与建档、旅程状态  
2. 按「客户姓名 + 手机后四位」维护/核验可关联订单  
3. 配置欢迎券与排除商品  
4. 维护商品是否代餐、适合哪类复购分支  
5. 调整 28 天方案与 Day28 A/B/C 规则  
6. 客服协助「关联不上」等人工处理  

---

## 2. 角色与能做什么

| 角色 | 能做的事 |
|------|----------|
| 管理员 | 全部配置与敏感操作 |
| 运营 | 商品、券、内容话术、复购规则、旅程模板 |
| 客服 | 查看用户；协助关联；备注；有限次补发/作废欢迎券（需确认） |
| 商品同学 | 维护商品信息、代餐标记、复购角色 |
| 合规 | 审核高风险话术与对外健康声明 |

敏感操作（改方案状态、绑单、发券、改复购规则）需可追溯：谁、何时、改了什么、原因。

## 3. 总览台

建议一眼看到：

- 今日注册、关联成功、内购待收货、确认收货开通数  
- 欢迎券发放 / 使用 / 使用率  
- Day28 到达与 A/B/C 主按钮点击  
- 关联失败常见原因  

点击可进入对应明细列表。

## 4. 用户

### 4.1 列表筛选

注册时间、地区、产品态（无产品 / 非代餐 / 代餐进行中 / 待收货）、是否已建档、券状态等。

### 4.2 详情应能看到

- 当前产品态、旅程第几天、提醒时间  
- 已关联订单与内购单、确认收货时间  
- 优惠券状态  
- 近几日打卡摘要（只读）  
- 若有高风险标记，仅合规/客服可见处理备注  
- 人工操作历史  

### 4.3 客服可操作（需二次确认）

- 协助绑定订单（预览命中结果后确认）  
- 对「待确认收货」用户代为确认并开通 Day 1（例外场景）  
- 有限次补发或作废欢迎券  
- **不可**直接篡改用户体重等打卡原始数值（防作弊）；异常走工单  

---

## 5. 订单

### 5.1 订单业务字段（概念）

订单号、客户姓名、手机（展示脱敏、检索可用后四位）、商品、是否代餐、支付与履约状态、下单时间、地区、是否已被用户关联。

### 5.2 与 App 关联规则一致

| 规则 | 说明 |
|------|------|
| 检索键 | 客户姓名 + 手机后四位 |
| 时间窗 | 默认近 180 天 |
| 允许状态 | 已付款 / 已发货 / 已签收等有效状态 |
| 多笔订单 | 返回列表；含代餐则优先开通 28 天方案 |
| 防刷 | 限制同一用户短时间反复试错次数 |

### 5.3 关联尝试记录

每次用户尝试关联，后台可查：时间、结果（成功 / 未找到 / 过期 / 状态不符）、命中几笔。供客服排障。

### 5.4 客服协助关联流程

```
用户说关联不上
  → 打开该用户
  → 协助关联：输入姓名 + 后四位
  → 预览将绑定的订单与商品
  → 确认绑定
  → 用户在 App 中看到产品态更新
```

## 6. 优惠券

### 6.1 可配置项

- 名称、面额、币种  
- 有效天数（如注册起 30 天）  
- 适用全店或排除某些商品  
- 是否注册自动发放  
- 与满减等活动的叠加规则说明  

### 6.2 与 App 一致的核销业务

下单确认时自动选合适券，用户可取消；支付成功才算使用；退款规则与 `规则-开通复购与优惠券.md` 一致。

### 6.3 运营日常

- 查看发放与使用明细  
- 调整排除商品名单  
- 停用某券模板（不影响已发出未用完的券的处理策略需写清）  

---

## 7. 商品

除名称、价格、图文、上下架外，与方案相关的业务标记：

| 标记 | 业务含义 |
|------|----------|
| 是否代餐 | 关联成功后是否开通 28 天 Slim Journey |
| 复购角色 | 适合作为：当前续购 / 维稳下一程 / 替代尝试 / 普通商品 |
| 是否可用欢迎券 | 是否允许欢迎券（可与排除名单配合） |

Day28：

- **A** 偏向「当前代餐续购」类商品  
- **B** 偏向「维稳 / 防护」类  
- **C** 偏向「替代尝试」类  

运营可在复购规则里指定各分支主推商品。

---

## 8. 方案与 Day28 规则

### 8.1 28 天模板（业务）

- 总天数 28  
- 阶段名称与文案（启动 / 适应 / 稳定 / 收官等）  
- 里程碑日（如 7 / 14 / 21 / 28）  
- 每日默认任务：体重、代餐、饮水、睡眠等  

### 8.2 Day28 分流可调参数

- 「有效果」最少下降多少  
- 「达标」与目标体重允许多少误差  
- A/B/C 各自主推哪个商品  

应提供「试算」：输入示例体重变化，预览会落在哪一分支。

### 8.3 内购开通策略（业务开关）

- 默认：支付后待确认收货；用户确认（或物流显示已签收）后开始 Day 1。  
- 可配置是否允许用户手动确认收货（默认允许）。

## 9. 内容配置

### 9.1 Sunny 话术区块

首次介绍、产品介绍、建档各步、Day1 引导、高风险回复等；按地区/语言维护。

### 9.2 Journey 每日建议

- 未建档用户：App **不展示**建议（后台预览需标明）。  
- 已建档：按代餐 / 非代餐 / 无产品套用不同建议模板。  
- 建议中的热量、饮水等应与评分目标同一套业务口径。  

---

## 10. 风控与合规（业务）

- 关联尝试过于频繁时的限制策略说明  
- 未满 18 岁拦截文案  
- 高风险场景标准回复（由合规审定）  
- 用户投诉与人工复核队列  

---

## 11. 关键业务操作流

### 11.1 新欢迎券

新建券模板 → 设排除商品 → 打开「注册自动发放」→ 新用户注册即得 → 下单页自动选用（可取消）。

### 11.2 改 Day28 维稳主推品

打开 Day28 规则 → 改 B 分支主推商品 → 保存 → 之后到达 Day28 的用户主按钮指向新品。

### 11.3 签收后自动开始（若启用）

物流显示已签收 → 若该单为 App 内购且仍待确认 → 自动开始 Day 1 → 通知用户。  
若关闭自动开始，则只靠用户或客服确认。

---

## 12. 验收要点

- [ ] 姓名 + 后四位能协助绑定测试订单  
- [ ] 注册自动发券；下单默认勾选；取消后支付券仍可用  
- [ ] 支付成功后券为已使用  
- [ ] 修改 B 分支主推品后，报告主按钮方向正确  
- [ ] 协助关联与改状态有操作痕迹  
- [ ] 未建档用户不会在 App 看到每日建议
