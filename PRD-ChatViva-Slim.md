## 1. Summary

luckdate（ChatViva Slim）以 AI 助手 **Sunny** 陪伴用户完成健康瘦身习惯。用户通过「客户姓名 + 手机后四位」关联外渠订单，或在 Mall 内购并 **确认收货** 后，开启 **28 天代餐 Slim Journey**。每日在 Journey（Ritual）查看建议与打卡，在 Plan（二级页）看方案进度，在 Sunny 对话记录与引导。Day 28 按效果分流引导复购。

## 3. Background

- 演示版已打通主开通链路；本文以 **完整产品交互** 为准，不因 Demo 省略步骤。
- 信息架构已定稿：**底栏 4 项** Sunny / Journey / Mall / Me；**Plan 不进底栏**，从 Journey 卡片、Me、Sunny 行动按钮进入。
- Sunny 产品介绍已并入 **首次进入 Sunny 聊天**，不再使用独立 3 页介绍主路径。

## 4. Objective

**目标：** 已购用户在 28 天内形成 Ritual 习惯，并在结束后被清晰引导到下一程复购；未购用户可探索并完成首次购买开通。

**Key Results（建议，基线待填）：**

| KR | 指标 | 目标 |
|----|------|------|
| KR1 | 注册后完成订单关联或内购确认收货 | ≥ 40% / 30 天 |
| KR2 | 代餐用户 Day7 Ritual ≥4/7 天 | ≥ 50% |
| KR3 | Day28 点击复购主 CTA | ≥ 25% |
| KR4 | 欢迎券核销率（30 天内） | ≥ 15% |

## 5. Market Segment(s)

| 分群 | Job | 方案形态 |
|------|-----|----------|
| 外渠已购代餐 | 关联订单立刻开始 | 28 天 Slim Journey |
| App 内购代餐 | 买完等到货再开始 | 待确认收货 → Day 1 |
| 外渠非代餐 | 每日服用提醒 | Product Care（无完整 28 天里程碑） |
| 未购 | 先聊再买 | 基础记录 + Plan 购买引导 |
| 未成年 / 高风险 | 安全退出 | 拦截继续开通（合规必做） |

---

## 6. Value Proposition(s)

- 订单资格与方案硬绑定，减少「买了不会用」。
- Sunny 对话 = 建档 + 日常打卡同一入口。
- Journey 基于基础信息给出每日摄入建议（无基础信息则不推荐）。
- Day28 按效果 A/B/C 分流复购，而非笼统「逛商城」。
- $5 欢迎券在下单时自动选用，可取消。

## 7. Solution — 完整功能交互

### 7.0 主导航

| 入口 | 类型 | 职责 |
|------|------|------|
| **Sunny** | 底栏 → 全屏聊天（无底栏） | 介绍、建档、日常记录、行动跳转 |
| **Journey** | 底栏（页面为 Ritual） | 活力总览、**每日建议**、打卡入口、Plan 入口卡 |
| **Mall** | 底栏 | 商品列表 / 筛选 / 详情 / 下单 |
| **Me** | 底栏 | 订单、优惠券、提醒、确认收货、Plan 快捷入口 |
| **Plan** | **二级页，不进底栏** | 未开通 / 待收货 / 进行中 / 非代餐护理 |

---

### 7.1 注册与新手引导（必须闭环）

```
开屏（约 2s）
  → 引导页 Feel Alive
      → Log in → 登录 → Journey（回访）
      → Start My Journey → 注册
  → 注册成功：发放 $5 欢迎券（30 天）
  → 关联订单页（客户姓名 + 手机后四位，可 Skip）
  → 首次进入 Sunny 聊天：
      · Sunny 自我介绍 + 能力说明（原「Sunny 介绍」内容）
      · 若已关联：产品介绍卡 + 方案邀请
      · 建档问答：隐私 → 年龄 → 身高 → 体重 → 目标 → 餐次 → 提醒
  → 建档完成 → 可进主壳；代餐已开通则引导 Day 1 打卡
```

**闭环检查清单：**

| 步骤 | 成功终态 | 失败 / 旁路 |
|------|----------|-------------|
| 注册 | 已登录 + 券已发 + 进入关联订单 | 表单校验失败留在注册页 |
| 关联成功（代餐） | 方案 active Day1 + Sunny 产品介绍 + 建档 | — |
| 关联成功（非代餐） | Product Care + Sunny 建档 | — |
| 关联失败 | 提示联系客服 + 可继续 Skip / 重试 | 不阻断注册 |
| Skip | 无产品 + Sunny 建档 + Plan 购买引导 | — |
| 建档完成 | `onboardingComplete=true`；Journey 出现每日建议 | 未成年拦截不可完成开通 |
| 内购 | 待确认收货；多入口 Confirm → **Plan 进行中** | 可取消支付（Demo 为确认前取消） |

**不做：** 独立 Sunny 3 页介绍作为主路径；注册成功礼券页可为旁路，主路径注册后直接关联订单并在 Mall/Me 展示券。

---

### 7.2 关联订单

**检索键：** **客户姓名（收件人姓名）+ 手机号后四位**（双重校验，防穷举；生产加限频）。

| 结果 | 交互 |
|------|------|
| 命中代餐 | 立即开通 28 天方案 → Sunny 产品介绍 → 建档 |
| 命中非代餐 | Product Care → Sunny 建档 |
| 多产品 | 列表展示，代餐优先决定是否进 Slim Journey |
| 未命中 | 文案提示联系客服；可继续 Skip 走无产品路径 |
| Skip | 无产品模式 |

**不做：** 同步私域标签 / 私域聊天历史。

**演示：** 姓名 `meal`/`solar` → 代餐；手机 `0000` → 失败；其他 → 随机 1–3 个演示商品。

---

### 7.3 优惠券

| 规则 | 说明 |
|------|------|
| 发放 | 注册成功即发 $5，全店通用（排除 SKU 运营配置），30 天有效 |
| **核销路径** | **商品下单（Checkout）时**：系统判断是否有合适券 → **自动勾选使用** → 用户可 **取消勾选** → 确认支付后标记 `used` |
| 查看 | Me → Coupons：状态 unused/used/expired、剩余天数；可去 Mall |
| 退款 | 生产：退款后退券（仍有效期内可用） |

---

### 7.4 App 内购与确认收货

```
Mall → 商品详情 → Buy Now
  → Checkout（自动选券，可取消）
  → 支付成功 → 待确认收货
  → 用户在 Plan / Me / Journey 待收货卡 / Plan Intro 点 Confirm Receipt
  → 开通 Day 1 → **跳转 Plan 进行中页**
```

**原则：** 购买成功 ≠ 方案开始；确认收货后才 Day 1。

---

### 7.5 Journey（Ritual）每日建议

**条件：** 用户已完成建档（有基础信息：年龄/身高/体重/目标等）。  
**无基础信息：** **不展示**推荐模块。

**有基础信息时展示：**

- 建议每日热量、蛋白质、饮水量  
- 1–3 条基础习惯建议（按产品态：代餐替换餐次 / 非代餐服用提醒 / 无产品先记录）

---

### 7.6 Plan（二级）

三种核心态：

1. **未开通：** Buy Product / Provide Order / Browse Mall  
2. **待确认收货：** Confirm Receipt & Start Plan → Day 1  
3. **进行中：** Day x/28、今日任务、阶段叙事  

非代餐：Product Care（服用提醒 + 可升级到代餐商城）。

---

### 7.7 Sunny 聊天

- 首次进入：能力介绍 +（可选）产品介绍 + 建档  
- 日常：饮水/饮食/睡眠/运动/体重/产品使用意图识别  
- 行动按钮：Go to Ritual / View My Plan / Log Water 等  
- Day ≥ 28：Journey Complete 弹窗 → 报告页  

---

### 7.8 Day 28 下一程（复购分流）

按体重趋势（首末）与目标体重判定：

| 分支 | 条件 | 引导 |
|------|------|------|
| **A** | 有效果（体重下降）但 **未达目标** | 继续购买 **当前代餐产品**（Solar Protein） |
| **B** | 有效果且 **已达目标** | 购买 **下一阶段** 维稳/防护向产品（如 Youth Solar / Longevity / Recovery） |
| **C** | **无效果**（持平或反弹） | 引导 **替代产品**（如 Active Boost / Sun Femme 等） |

报告页主 CTA 直达对应商品详情；次 CTA 可进 Mall 浏览更多。

---

### 7.9 非代餐路径（完整描述）

此前「半闭环」指：已有 Product Care，但升级深度浅。完整规格：

- 每日提醒服用；可在 Sunny/Plan 记录 taken  
- 可浏览 Mall 升级 Solar Protein → 走内购确认收货或再次关联  
- **不**展示 28 天里程碑环  

---

### 7.10 风险与合规（上线门禁）

- 建档年龄 Under 18：拦截，不可开通 Slim Journey  
- 孕哺 / 自伤等高风险表达：安全回复 + 停止方案推荐（生产接客服）  
- L0–L4 完整医学分层可分期；**未成年硬拦截必须有**

---

### 7.11 Assumptions

1. Solar Protein 为 Slim Journey 资格 SKU（短期）。  
2. 确认收货在物流 API 到位前允许用户手动确认。  
3. 券排除 SKU、美墨订单规则由运营后台配置。  
4. Day28 A/B/C 判定以体重趋势为主（后续可加 Ritual 完成率加权）。
