# luckdate（ChatViva Slim）APP 功能交互说明（完整版）

> 本文描述用户可见的功能与交互。Demo 中登录/支付/订单为模拟，但交互路径按完整产品描述。  
> 界面文案为英文；本说明为简体中文。  
> **同步：** PRD V1.2 · 2026-07-24

---

## 第一部分：核心流程

### 1. 产品目标与闭环

1. 新用户完成注册与 Sunny 建档  
2. 具备产品资格后开启 Slim 方案（关联订单立即 Day1，或内购确认收货后 Day1）  
3. 每日在 Journey / Sunny 记录；Plan 查看进度  
4. Day28 按效果 A/B/C 引导复购  

**原则：先有产品资格，再进入 28 天方案。**

### 2. 主导航

| 入口 | 说明 |
|------|------|
| **Sunny** | 全屏对话（无底栏）；含首次介绍与建档 |
| **Journey** | 活力总览、每日建议、打卡、Plan 入口卡 |
| **Mall** | 商品浏览与购买 |
| **Me** | 订单、优惠券、提醒、确认收货 |
| **Plan** | **二级页，不进底栏** |

### 3. 新用户主路径

```
开屏 → 引导页
  → Start My Journey → 注册（发 $5 券）
  → 关联订单（客户姓名 + 手机后四位，可 Skip）
  → Sunny 聊天首次进入：能力介绍 +（若已关联）产品介绍 + 建档问答
  → 主壳
```

回访：`Log in` → Journey。

### 4. 方案开通

| 链路 | Day1 时机 |
|------|-----------|
| 姓名+后四位关联代餐 | **立即** |
| Mall 购买 Solar Protein | Checkout（券自动选用可取消）→ 待收货 → **Confirm → 跳转 Plan** |

### 5. Day28 复购

| 分支 | 条件 | 引导 |
|------|------|------|
| A | 有效果未达目标 | 续购当前代餐 |
| B | 有效果且达目标 | 维稳/防护下一阶段产品 |
| C | 无效果 | 替代产品 |

### 6. Journey 每日建议

- 已建档：展示热量/蛋白/饮水与基础建议  
- 未建档：**不展示**  

### 7. 验收路径

- A：注册 → 关联代餐 → Sunny 建档 → Day1  
- B：Skip → 建档 → Mall 下单（用券）→ Confirm → Plan  
- C：登录回访 → Journey  
- D：Day28 → A/B/C 商品详情  
- E：无建档无建议；完成建档后出现建议

## 第二部分：全功能交互（摘要更新）

### 关联订单

输入：**Recipient name** + **手机后四位**。失败可继续；Skip 进无产品。

### 优惠券

Me 可查看。核销仅在 **商品 Checkout**：系统自动选合适券，用户可取消；支付成功标记 used。

### 确认收货

Plan / Me / Journey 待收货卡 / Plan Intro → 确认后 **进入 Plan 进行中**。

### Sunny

无独立 3 页介绍主路径；介绍内容在首次聊天中。

### Plan

不进底栏；从 Journey「Your Plan」卡、Me Quick Menu、Sunny 行动进入。

### 演示与占位

登录/订单/支付仍为 Demo Mock；流程交互已按上表闭环。分享/搜索等部分控件仍可为占位。
