# 规则 — 每日记录与评分

> 产品：luckdate · 版本 V2.1 · 2026-07-24  
> 范围：记什么、怎么算分、怎么展示。不含技术实现。  
> 对应主文档：`PRD-luckdate.md`  
> 对应界面：Journey 分数环与五大维度（**不含冥想 Mindfulness**）

---

## 1. 原则

- **行为优先：** 奖励「有没有记录」，而不是奖励体重下降或运动量大小。  
- **不惩罚情绪：** 心情差、睡眠差不扣分；只触发 Sunny 关怀。  
- **低焦虑：** 不强调断签、失败、排名；用分数与稳定性表达成长。  
- **可解释：** 每个分数都能说清来自「有记录 / 无记录」或近 7 日连续情况。  

分数用于「我在坚持」的反馈，**不用于诊断健康、预测减重效果或评价用户好坏。**  
**体重数值升/降不改变任何当日分数**，只影响「是否已记录体重」。

---

## 2. 每日可记录项

| 记录项 | 用户可能怎么完成 | 评分只看 | 备注 |
|--------|------------------|----------|------|
| 产品使用 | Journey / Plan / Sunny | **有记录 / 无记录** | 不区分吃全/吃一半；记下即算有记录 |
| 饮水 | Journey / Sunny | **有记录 / 无记录** | 任意大于 0 的饮水量即算有记录；是否达目标不进该项分 |
| 体重 | 手动或秤 | **有记录 / 无记录** | 数值本身不加分不扣分 |
| 状态反馈 | Journey / Sunny | **有记录 / 无记录** | 有心情或精力等任一标签即算；负面不扣分 |
| 睡眠 | Journey / Sunny | **有记录 / 无记录** | 有小时数或质量感受任一即算 |
| 饮食（可选） | Sunny / 打卡 | 计入 Nutrition 维度有记录 | 有一餐或摄入记录即可 |
| 运动（可选） | Sunny / 打卡 | 计入 Exercise 维度有记录 | 有时长或消耗记录即可 |
| 旅程日 | 系统按用户当地日 | — | Day 1～28，按时区算「今天」 |

饮水目标（默认约 2000ml）仍可用于 Journey **建议文案**与进度提示，**不再用于饮水分计分**。

## 3. 对外指标一览（对齐 Journey 五大指标）

Journey 页面对外展示 **5 个维度分**（已去掉冥想）+ **1 个综合生命力分**。全部为 **0～100 的分数**，不使用百分比展示。

| 指标 | 范围 | 业务含义（怎么判定） | 对用户怎么说 |
|------|------|----------------------|--------------|
| **每日生命力分 Daily Vitality** | 0～100 | 当日加权综合（见第 4 节） | 今天的生命力状态 |
| **Nutrition 营养** | 0～100 | 今日是否有 **产品或饮食** 记录 | 营养记录 |
| **Exercise 运动** | 0～100 | 今日是否有 **运动** 记录 | 运动记录 |
| **Sleep 睡眠** | 0～100 | 今日是否有 **睡眠** 记录 | 睡眠记录 |
| **Hydration 饮水** | 0～100 | 今日是否有 **饮水** 记录 | 饮水记录 |
| **Habits 习惯（温和稳定性）** | 0～100 | 近 7 日完成度均值 + 连续完成加分（见 5.6） | 习惯稳定性 |

**说明：**

- 原「仪式完成度 %」「饮水进度 %」不再作为对外主指标；完成情况并入上述 **分数**。  
- 原「状态反馈 / 产品仪式」仍参与 **生命力加权**（第 4～5 节），状态反馈不再单独作为 Journey 第六环（冥想已去掉）。  
- Journey 五大环均显示整数分（如 `80`），评级文案可用 Good / Fair 等，但主数字是分数不是 %。

## 4. 每日生命力分（加权口径）

$$
\text{Daily Vitality} = \mathrm{round}\big(
  S_{产品}\times0.25 +
  S_{体重}\times0.25 +
  S_{饮水}\times0.15 +
  S_{状态}\times0.15 +
  S_{睡眠}\times0.10 +
  S_{稳定}\times0.10
\big)
$$

各 $S$ 均为 **0 或 100**（稳定性见 5.6，可为 0～100 之间整数）。展示时四舍五入为整数。

| 维度 | 权重 | 子分 $S$ 来源 |
|------|------|----------------|
| 产品仪式 | **25** | 有产品记录 → 100；无 → 0 |
| 体重记录 | **25** | 有体重记录 → 100；无 → 0 |
| 饮水 | **15** | 有饮水记录 → 100；无 → 0 |
| 状态反馈 | **15** | 有状态记录 → 100；无 → 0 |
| 睡眠 | **10** | 有睡眠记录 → 100；无 → 0 |
| 温和稳定性 | **10** | 按 5.6 计算的 0～100 分 |

**示例：**  
五项当日均有记录（各 100），稳定性 80 →  
$25+25+15+15+10+8 = 98$ → 展示 **98**。

## 5. 各子项判定（具体分值）

以下五项 **只判有记录 / 无记录**，分值固定为 **100 或 0**。

### 5.1 产品仪式分 $S_{产品}$

| 用户情况 | 分值 |
|----------|------|
| 今日已记录产品使用（含 Sunny「喝了代餐」等） | **100** |
| 今日无任何产品使用记录 | **0** |

Sunny：有记录时肯定坚持；无记录时晚间轻提醒，不显示「失败」。

### 5.2 饮水分 $S_{饮水}$

| 用户情况 | 分值 |
|----------|------|
| 今日饮水量 > 0（任意单位换算后） | **100** |
| 今日无饮水记录 | **0** |

是否达到饮水目标 **不影响该项分数**；目标仅用于建议与文案提示。

### 5.3 体重记录分 $S_{体重}$

| 用户情况 | 分值 |
|----------|------|
| 今日已记录体重数值 | **100** |
| 今日未记录 | **0** |

体重升/降 **不改变** 本分。趋势仅用于报告与 Day28 复购。

### 5.4 状态反馈分 $S_{状态}$

| 用户情况 | 分值 |
|----------|------|
| 今日有心情 / 精力 / 压力等任一状态记录 | **100** |
| 今日无状态记录 | **0** |

负面状态仍给 **100**（只要有记录），并触发关怀，不扣分。

### 5.5 睡眠分 $S_{睡眠}$

| 用户情况 | 分值 |
|----------|------|
| 今日有睡眠小时数 **或** 睡眠质量感受 | **100** |
| 今日两者皆无 | **0** |

睡多久、自评好坏 **不改变** 本分；差睡眠由 Sunny 建议减轻任务，不惩罚分数。

---

### 5.6 温和稳定性分 $S_{稳定}$（Habits）

结合 **近 7 日完成度** 与 **连续完成天数**，给出 0～100 的具体分。

#### 5.6.1 何为「当日完成」

统计当日 5 项核心记录是否有记录：产品、饮水、体重、状态、睡眠。

| 判定 | 规则 |
|------|------|
| 当日完成 | 上述 5 项中，**有记录的项数 ≥ 3** |
| 当日未完成 | 有记录项数 ≤ 2 |

旅程第 1～6 天：只统计「已过去的自然日」，分母用实际天数，不用凑满 7。

#### 5.6.2 近 7 日完成度均值 $A$

$$
A = \mathrm{round}\left(\frac{\text{近 7 日内「当日完成」的天数}}{\text{可统计天数 } n}\times 100\right)
$$

- $n = \min(7,\ \text{用户进入旅程以来的天数})$，至少为 1。  
- $A$ 范围 0～100。

#### 5.6.3 连续完成加分 $C$

从 **今天** 往前数连续「当日完成」的天数（今天未完成则连续为 0）：

| 连续完成天数 | 连续分 $C$ |
|--------------|------------|
| 0 | **0** |
| 1 | **40** |
| 2 | **55** |
| 3 | **70** |
| 4 | **80** |
| 5 | **90** |
| ≥ 6 | **100** |

界面文案避免「断签」；可说「最近更稳了」「今天接上就好」。

#### 5.6.4 合成稳定性分

$$
S_{稳定} = \mathrm{round}\big(0.7\times A + 0.3\times C\big),\quad \text{限制在 } 0\sim100
$$

**示例：**

| 近 7 日完成天数 | $A$ | 当前连续 | $C$ | $S_{稳定}$ |
|----------------|-----|----------|-----|------------|
| 7/7 | 100 | 7 | 100 | **100** |
| 5/7 | 71 | 2 | 55 | $\mathrm{round}(49.7+16.5)=66$ → **66** |
| 3/7 | 43 | 0 | 0 | **30** |
| 0/7 | 0 | 0 | 0 | **0** |

## 6. Journey 五大维度分（与第 3 节一一对应）

五大环各自独立为 0～100，**均为有记录 100 / 无记录 0**（Habits 除外）。

| Journey 维度 | 分值规则 |
|--------------|----------|
| **Nutrition** | 今日有产品记录 **或** 有饮食记录 → **100**；否则 **0** |
| **Exercise** | 今日有运动记录 → **100**；否则 **0** |
| **Sleep** | 同 $S_{睡眠}$ → **100 / 0** |
| **Hydration** | 同 $S_{饮水}$ → **100 / 0** |
| **Habits** | 同 $S_{稳定}$（5.6）→ **0～100** |

**不展示：** Mindfulness / 冥想环。

综合生命力环使用第 4 节公式（含体重、状态等加权），与五大环并行展示：  
- 五大环：分项习惯是否记下  
- 生命力环：加权后的「今天整体节奏」

## 7. 展示文案方向（避免伤害）

| 生命力分区间 | 表达方向 | 避免 |
|--------------|----------|------|
| 90～100 | 节奏很好 | 「完美」「马上会瘦」 |
| 75～89 | 很稳定 | 「还差一点才合格」 |
| 50～74 | 慢慢接上，补记 1～2 项即可 | 「低分」「失败」 |
| 1～49 | 一小步也算，先记一件 | 「拖后腿」「不自律」 |
| 0 | 随时可以重新开始 | 「断签」「失败」 |

单维为 0 时，可用「Not logged / 今天还没记」，不用惩罚语气。

---

## 8. 体重趋势（报告用，不算日分）

| 概念 | 业务用法 |
|------|----------|
| 每日体重 | 只作记录 |
| 近几日平滑趋势 | 减少单日波动焦虑 |
| 与第 1 天对比 | 宜在接近结束的报告中温和展示 |
| 平台期 | 记录很全但体重少变时，解释波动，强调习惯 |
| 异常波动 | 变化过大 → 关怀与安全提示，而非庆祝 |

Day28 复购 A/B/C 使用体重趋势与目标，见 `规则-开通复购与优惠券.md`。

---

## 9. Journey 每日建议与评分的关系

- **建议模块：** 依赖建档；无建档不展示。可提示饮水目标，但 **达标与否不改饮水分**。  
- **评分模块：** 只认有没有记；没记则为 0，不羞辱用户。  

---

## 10. 验收要点

- [ ] 生命力权重：产品 25、体重 25、饮水 15、状态 15、睡眠 10、稳定 10  
- [ ] 产品 / 饮水 / 体重 / 状态 / 睡眠仅为 100 或 0  
- [ ] 饮水未达标但有记录 → 饮水分仍为 100  
- [ ] 体重升降不改变体重记录分  
- [ ] Journey 只展示五大维度 + 生命力，**无冥想**；全部为分数非 %  
- [ ] Habits / 稳定性按 $0.7A+0.3C$ 可手算复核  
- [ ] 文案无断签羞辱
