> **版本：** V1.0  
> **日期：** 2026-07-24  
> **读者：** 运营、客服、商品、研发

---

## 1. 后台目标

支撑 App 端完整闭环：

1. 用户与建档数据可查  
2. 外渠订单可按「客户姓名 + 手机后四位」检索关联  
3. 优惠券发放 / 核销 / 排除 SKU 可配  
4. 商品与「是否代餐 / 下一程角色」可配  
5. Day28 复购规则参数可调  
6. 客服可处理关联失败与人工开通

## 4. 用户 Users

### 4.1 列表

筛选项：注册时间、地区、产品态（无产品 / 非代餐 / 代餐进行中 / 待收货）、建档是否完成、券状态。

列：用户 ID、昵称、邮箱、地区、产品态、Journey Day、关联订单号、注册时间。

### 4.2 详情

Tab：

| Tab | 内容 |
|-----|------|
| 概览 | 状态机、建档字段、提醒时间 |
| 订单 | 已关联订单、内购单、确认收货时间 |

- **不可**直接改体重打卡原始数据（防造假）；异常走工单

## 5. 订单 Orders

### 5.1 订单列表

来源：电商 / 私域导入 / App 内购。

字段：订单号、客户姓名、手机号（脱敏显示，后四位可检索）、SKU、是否代餐、支付状态、发货/签收状态、下单时间、地区。

筛选：状态、是否已被 App 关联、代餐标记、时间窗（默认近 180 天）。

### 5.2 关联规则（与 App 一致）

App 调用：

```
POST /admin-api 对应服务
入参：recipient_name + phone_last4
```

后台配置：

| 配置项 | 说明 |
|--------|------|
| 时间窗 | 默认 180 天 |
| 允许状态 | 已付款 / 已发货 / 已签收 |
| 限频 | 同 IP / 同设备 / 同账号次数 |
| 多订单策略 | 返回列表；含代餐则优先开通 Slim |

### 5.3 关联查询日志

每次 App 检索记录：时间、用户、姓名哈希、后四位、结果码（成功/未找到/过期/状态不符）、命中订单数。供风控与客服排查。

### 5.4 人工协助关联

客服在用户详情点「协助关联」→ 输入姓名+后四位 → 预览命中 → 确认绑定。写审计日志。

## 6. 优惠券 Coupons

### 6.1 券模板

| 字段 | 示例 |
|------|------|
| 名称 | Welcome $5 |
| 面额 / 币种 | 5 USD |
| 有效期 | 注册起 30 天 |
| 适用范围 | 全店 |
| 排除 SKU | 多选（低毛利/临期等） |
| 叠加 | 可与满减叠加；不可与其他券 |
| 自动发放触发 | 注册成功 |

### 6.2 核销逻辑（与 App Checkout 一致）

1. 用户下单时，系统拉取该用户 **unused 且未过期** 且 **SKU 不在排除列表** 且 **订单金额 ≥ 面额** 的券。  
2. **默认自动选中最优一张**（面额最大；同额选最早过期）。  
3. 用户可在 Checkout **取消选用**。  
4. 支付成功 → 券 `used`，记录 order_id、used_at。  
5. 全额退款 → 券退回 `unused`（若仍在有效期内），否则 `expired`。

### 6.3 后台页面

- 模板列表 / 编辑  
- 发放明细（用户、状态）  
- 核销报表

## 7. 商品 Products

### 7.1 商品编辑

基础：名称、系列、价格、图、成分、用法、警示、上下架。

**与方案相关的关键字段：**

| 字段 | 作用 |
|------|------|
| `is_meal_replacement` | 关联后是否开通 28 天 Slim Journey |
| `journey_role` | `slim_core` / `maintain` / `alternative` / `general` |
| `eligible_for_welcome_coupon` | 是否允许欢迎券（细于排除列表时） |

### 7.2 与 Day28 分支映射

| 分支 | 默认推荐来源 |
|------|----------------|
| A 继续当前 | `journey_role = slim_core` 且与用户当前 SKU 同类 |
| B 维稳防护 | `maintain` 优先 |
| C 替代 | `alternative` 优先 |

运营可在「复购规则」覆盖默认 SKU。

## 8. 方案 Plans

### 8.1 Slim Journey 模板

- 总天数：28  
- 阶段：Launch / Adaptation / Stability / Completion（可改文案）  
- 里程碑日：7 / 14 / 21 / 28  
- 每日任务模板：体重 / 代餐 / 饮水 / 睡眠  

### 8.2 Day28 复购规则

可配置阈值（可视化表单）：

```
IF weight_loss >= X kg AND current_weight > target + tol
  → Path A → product_id = …
ELSE IF weight_loss >= X kg AND current_weight <= target + tol
  → Path B → product_id = …
ELSE
  → Path C → product_id = …
```

参数：`X`（默认 0.5kg）、`tol`（默认 0.5kg）。  
可预览：输入示例体重曲线 → 显示命中分支。

### 8.3 内购开通策略

固定：**支付成功 → awaiting_receipt；确认收货（或物流签收回调）→ active Day1**。  
后台可开关「允许用户手动确认收货」（默认开）。

## 9. 内容 Content

### 9.1 Sunny 话术

区块：首次介绍、产品介绍、建档各步、Day1 引导、高风险回复。  
支持按地区（US/MX）与语言版本。

### 9.2 Journey 每日建议模板

变量：`{calorie}` `{protein}` `{water}` `{meal_slot}` `{reminder_time}`。  

规则：

- `onboarding_complete = false` → App **不展示**建议模块（后台预览需标注）。  
- 完成建档后按产品态选择模板（代餐 / 非代餐 / 无产品）。

热量公式可配置为「固定目标」或「公式估算」（与 App `DailyAdviceHelper` 对齐）。

## 11. 关键后台交互流程

### 11.1 客服协助「关联不上」

```
用户反馈关联失败
  → 客服打开用户详情
  → 协助关联：输入姓名+后四位
  → 预览订单 → 确认绑定
  → App 用户刷新后看到产品 / 方案状态更新
```

### 11.2 运营配置新欢迎券

```
优惠券 → 新建模板 → 设排除 SKU → 启用「注册自动发放」
  → 新注册用户自动获得
  → Checkout 自动选用（用户可取消）
```

### 11.3 运营调整 Day28 维稳产品

```
方案 → Day28 规则 → Path B 主推商品改为 Longevity Solar
  → 保存 → App 报告页主 CTA 指向新商品
```

### 11.4 物流签收自动开通（生产）

```
仓储回调「已签收」
  → 若订单为 App 内购且 awaiting_receipt
  → 自动 confirm → Day1
  → 推送通知用户
```

（若关闭自动开通，则仅依赖用户手动确认。）

## 12. 权限与审计
