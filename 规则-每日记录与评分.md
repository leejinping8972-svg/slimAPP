# 规则 — 每日记录与评分

> 产品：luckdate · 版本 V2.0 · 2026-07-24  
> 范围：记什么、怎么算分、怎么展示。不含技术实现。  
> 对应主文档：`PRD-luckdate.md`

---

## 1. 原则

- **行为优先：** 奖励记录、饮水、产品使用与状态反馈，而不是奖励「体重下降」。  
- **不惩罚情绪：** 心情差、睡眠差不扣「人品分」；只触发 Sunny 关怀或减轻当日任务建议。  
- **低焦虑：** 不强调断签、失败、排名；用完成度、稳定性、里程碑表达成长。  
- **可解释：** 每个分数都能说清来自哪几项今日行为。  
- **可配置：** 权重、饮水目标、某日哪些任务必做，可由运营按旅程模板调整。

分数用于「我在坚持」的反馈，**不用于诊断健康、预测减重效果或评价用户好坏。**

---

## 2. 每日可记录项

| 记录项 | 用户可能怎么完成 | 业务取值（概念） | 备注 |
|--------|------------------|------------------|------|
| 是否使用产品 | Journey / Plan / Sunny | 已用 / 部分 / 有原因跳过 / 未记录 | 按当日产品计划判断，不评价医学效果 |
| 饮水量 | Journey / Sunny | 毫升或盎司累计 | 对照个人饮水目标 |
| 饮水目标 | 个人设置 | 默认约 2000ml，可改 | 过低或过高建议二次确认 |
| 体重 | 手动或秤 | 数值 + 单位 | **数值本身不加分不扣分** |
| 是否记录体重 | — | 已记录 / 有原因跳过 / 未记录 | 只看「有没有记」 |
| 状态感受 | Journey / Sunny | 心情、精力、饥饿、压力等标签 | 负面不扣核心分 |
| 睡眠 | Journey / Sunny | 小时数 + 质量感受 | 用于睡眠分与关怀 |
| 旅程日 | 系统按用户当地日 | Day 1～28 | 按时区计算「今天」 |

不同旅程模板可增减「今日必做任务」清单。

## 3. 对外指标一览

| 指标 | 大致范围 | 业务含义 | 对用户怎么说 |
|------|----------|----------|--------------|
| 每日生命力分 Daily Vitality | 0～100 | 当天多维度加权 | 今天的生命力状态 |
| 仪式完成度 Ritual Completion | 0～100% | 今日任务完成数 / 应做数 | 今日仪式完成情况 |
| 饮水进度 | 0～100% | 已喝 / 目标，封顶 100% | 饮水进度 |
| 产品仪式分 | 0～100 | 是否按计划用产品 | 产品仪式 |
| 体重记录分 | 0～100 | 是否记录体重 | 体重记录 |
| 状态反馈分 | 0～100 | 是否完成状态反馈 | 状态反馈 |
| 睡眠分 | 0～100 | 记录完整度 + 时长/质量温和分 | 睡眠恢复感 |
| 温和稳定性 | 0～100 | 近 7 天完成度均值 | 最近是否更稳定 |

**仪式完成度 ≠ 生命力分：**  
前者适合「今天 4/5」；后者适合综合状态环。

## 4. 每日生命力分（默认口径）

默认加权（总和 100）：

| 维度 | 权重 | 怎么理解 |
|------|------|----------|
| 产品仪式 | 25 | 按计划用了产品得高分；有原因跳过不完全归零 |
| 饮水 | 25 | 相对目标的完成比例，超过目标不额外加分 |
| 体重记录 | 15 | 只奖励「记下了」 |
| 状态反馈 | 15 | 完成反馈得分；负面情绪不扣分 |
| 睡眠 | 10 | 有记录，并结合时长与自评质量 |
| 近 7 天稳定性 | 10 | 最近几天平均完成情况 |

展示时四舍五入为整数。运营可调整权重，但需保持可解释。

## 5. 各子项判定

### 5.1 产品仪式

| 用户情况 | 分数倾向 | Sunny 方向 |
|----------|----------|------------|
| 已按计划使用 | 满分 | 肯定坚持，不夸大效果 |
| 只用了一部分 / 延后 | 中等偏高 | 认可部分完成，给温和补救建议 |
| 主动跳过并说明原因 | 较低但不为零 | 不责备，可问是否要调计划 |
| 未记录 | 零 | 晚间轻提醒，不显示「失败」 |

### 5.2 饮水

- 进度 = 已喝量 / 目标，最高按 100% 计。  
- 默认目标约 2000ml（或地区习惯单位等价）；用户可改。  
- 目标过低或过高建议确认，避免分数失真或过度饮水导向。  
- 未达目标仍可算「今日已记录饮水」（见仪式完成度）；未达标只影响饮水分。

### 5.3 体重记录

| 情况 | 分数倾向 |
|------|----------|
| 今日有记录 | 满分 |
| 跳过但选了原因 | 中等 |
| 未记录 | 零 |
| 体重升或降 | **不改变当日分数** |

体重趋势用于报告与 Day28 复购判断，不直接打今日生命力分。

### 5.4 状态反馈

- 选了心情，并补充精力/饥饿等至少一项 → 满分倾向。  
- 只选心情，或只在聊天里描述并由系统理解 → 中高分。  
- 未记录 → 零。  
- 负面状态 → **不扣分**，触发关怀。

### 5.5 睡眠

| 情况 | 分数倾向 |
|------|----------|
| 约 ≥7 小时且自评尚可/良好 | 高 |
| 约 6～7 小时或自评一般 | 中高 |
| ＜6 小时或自评差 | 中等（不惩罚人格） |
| 只记质量未记小时 | 中等 |
| 未记录 | 零 |

睡眠差时，Sunny 宜建议减轻当日压力或选更简单任务，而不是指责。

### 5.6 温和稳定性

- 用最近最多 7 天的仪式完成度平均值。  
- 旅程前几天按已有天数平均。  
- 漏记只影响平均值；界面避免「断签 X 天」恐吓式表达。

## 6. 仪式完成度

默认「今日应做」任务示例：产品回应、饮水有记录、体重回应、状态反馈、睡眠反馈（模板可改）。

- 有原因的跳过，可算「今日已回应该任务」，但产品分仍较低。  
- 饮水只要有记录即可算该任务完成；是否达标另算饮水分。

示例：五项都应做且都已回应 → 完成度 100%；饮水 1600/2000 仍算饮水任务完成，饮水分约 80%。

## 7. 展示文案方向（避免伤害）

| 生命力分大致区间 | 表达方向 | 避免 |
|------------------|----------|------|
| 90～100 | 节奏很好 | 「完美」「马上会瘦」 |
| 75～89 | 很稳定 | 「还差一点才合格」 |
| 50～74 | 慢慢接上，补一个最小任务 | 「低分」「失败」 |
| 1～49 | 一小步也算 | 「拖后腿」「不自律」 |
| 0 | 随时可以重新开始 | 「断签」「失败」 |

---

## 8. 体重趋势（报告用，不算日分）

| 概念 | 业务用法 |
|------|----------|
| 每日体重 | 只作记录 |
| 近几日平滑趋势 | 减少单日波动焦虑 |
| 与第 1 天对比 | 宜在接近结束的报告中温和展示 |
| 平台期 | 完成率高但体重少变时，解释波动，强调习惯 |
| 异常波动 | 单日变化过大或下降过快 → 关怀与安全提示，而非庆祝 |

Day28 复购 A/B/C 使用体重趋势与目标，见 `规则-开通复购与优惠券.md`。

---

## 9. Journey 每日建议与评分的关系

- **建议模块：** 依赖建档基础信息；无建档则不展示建议。  
- **评分模块：** 依赖当日记录；无记录则分数低或为零，但不羞辱用户。  
- 建议中的热量/饮水目标，应与评分用的目标一致（用户改目标后两边同步理解）。

---

## 10. 验收要点

- [ ] 体重升降不改变当日生命力分，只影响是否「已记录」  
- [ ] 负面心情不扣核心分  
- [ ] 饮水未达标仍可算仪式任务完成  
- [ ] 文案无断签羞辱  
- [ ] 未建档不出现每日建议；已建档出现
