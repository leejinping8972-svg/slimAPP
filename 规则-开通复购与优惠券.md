# 规则 — 开通、复购与优惠券

> 产品：luckdate · 版本 V2.0 · 2026-07-24  
> 范围：业务规则与用户交互。不含技术实现。  
> 对应主文档：`PRD-luckdate.md`

---

## 1. 用户产品态（业务状态）

| 状态 | 含义 | 用户主要看到 |
|------|------|--------------|
| 无产品 | 未关联成功 / 跳过 / 无有效资格 | Plan 购买引导；可聊天与基础记录 |
| 待确认收货 | 已在 App 内购买代餐，货未确认 | 多处「确认收货」入口 |
| 代餐进行中 | 28 天 Slim Journey 已开始 | Journey / Plan 显示 Day x/28 |
| 非代餐护理 | 关联到非代餐商品 | 每日服用提醒；无完整 28 天里程碑 |

同一用户可从「无产品」经购买或关联进入后两种状态。

## 2. 关联订单

### 2.1 何时出现

- 新用户注册成功后 **必须经过** 关联订单页（可跳过）。  
- 已登录用户可从 Me「订单」再次进入关联。

### 2.2 用户填写

| 项 | 规则 |
|----|------|
| 客户姓名 | 订单上的收件人/客户姓名，必填 |
| 手机号后四位 | 4 位数字，必填 |

用姓名 + 后四位共同查找，降低误绑与乱试。

### 2.3 结果处理

| 结果 | 业务处理 |
|------|----------|
| 找到代餐类商品 | **立即开通** 28 天方案（Day 1）→ 进入 Sunny（产品介绍 + 建档） |
| 只找到非代餐 | 进入产品护理模式 → Sunny 建档 |
| 找到多个商品 | 展示列表；只要含代餐，按代餐开通 28 天方案 |
| 找不到 | 提示「未找到订单，请联系客服」；可重试或跳过，**不阻断**后续 |
| 用户点跳过 | 无产品模式，进入 Sunny 建档 |

### 2.4 订单有效性（业务要求）

- 仅「已付款 / 已发货 / 已签收」等有效履约状态可关联。  
- 建议仅接受近 **180 天** 内订单；过期提示联系客服。  
- **不做：** 同步私域标签、私域聊天记录。

### 2.5 演示辅助（非生产规则）

演示环境可用约定姓名（如 meal/solar）模拟代餐成功；手机 `0000` 模拟失败。正式环境以真实订单为准。

## 3. App 内购买与确认收货

### 3.1 购买流程

```
Mall → 商品详情 → 立即购买
  → 确认订单页（见优惠券规则）
  → 支付成功
  → 状态变为「待确认收货」
  → 方案尚未开始
```

### 3.2 确认收货

- **原则：支付成功 ≠ 方案开始；确认收货后才进入 Day 1。**  
- 入口：Plan、Me、Journey 待收货提示、方案介绍页。  
- 用户确认后：开通 28 天方案，并 **进入 Plan 进行中页面**。

### 3.3 资格商品

短期内以约定代餐商品（如 Solar Protein）作为开通 28 天方案的资格商品；其他商品不自动开通完整里程碑旅程。

## 4. 优惠券

### 4.1 发放

| 项 | 规则 |
|----|------|
| 触发 | 注册成功即发放 |
| 面额 | $5（或运营配置的等值欢迎券） |
| 有效期 | 自发放起 30 天 |
| 范围 | 默认全店；部分商品除外（运营维护名单） |
| 叠加 | 可与满减类活动叠加；不可与其他优惠券叠加（若运营另有规定以配置为准） |

### 4.2 核销路径（唯一正式路径）

1. 用户进入商品下单确认页。  
2. 系统判断是否存在「未使用、未过期、适用于该商品、订单金额满足条件」的券。  
3. **默认自动选中** 合适券（多张时选优惠最大；相同则选更早过期）。  
4. 用户可 **取消勾选** 不用券。  
5. 支付成功后，券变为「已使用」。  
6. 若用户取消勾选后支付，券保持「未使用」。

### 4.3 查看

Me → 优惠券：展示面额、状态（未使用 / 已使用 / 已过期）、剩余天数；可引导去商城。

### 4.4 退款

订单全额退款后：若仍在有效期内，券退回「未使用」；否则记为过期。

## 5. Day 28 复购分流

### 5.1 触发

用户到达或完成第 28 天（代餐旅程）→ 完成提示 → 报告页 → 复购引导。

### 5.2 判定（业务）

比较旅程开始体重与近期体重，并对照用户目标体重：

| 分支 | 条件 | 用户引导 |
|------|------|----------|
| **A** | 体重有下降（有效果），但仍高于目标 | 继续购买 **当前代餐产品** |
| **B** | 体重有下降，且已达到或低于目标 | 购买 **下一阶段** 维稳 / 防护向产品 |
| **C** | 体重基本无下降或上升 | 引导购买 **替代产品** |

默认阈值参数（可由运营调整）：下降至少约 0.5kg 视为「有效果」；与目标相差约 0.5kg 内可视为「达标」。

### 5.3 页面行为

- 报告展示完成概况与 Sunny 寄语。  
- **主按钮** 进入该分支推荐的商品详情。  
- **次按钮** 可进入商城浏览更多。  
- 「稍后再说」可关闭；建议日后可再次提醒（运营可配间隔，如 7 天）。

## 6. 三种产品态下的差异（摘要）

| 模块 | 代餐进行中 | 非代餐护理 | 无产品 |
|------|------------|------------|--------|
| Plan | Day x/28 任务 | 服用提醒 | 购买 / 关联引导 |
| Journey 建议 | 含代餐餐次建议 | 含服用提醒建议 | 鼓励先记录与逛商城（需已建档） |
| 28 天里程碑 | 有 | 无 | 无 |
| Day28 复购 | 走 A/B/C | 不走完整毕业仪式 | 无 |

## 7. 验收要点

- [ ] 姓名 + 后四位可成功 / 失败 / 跳过，失败不阻断  
- [ ] 代餐关联后立即 Day 1  
- [ ] 内购后必须确认收货才 Day 1，确认后进入 Plan  
- [ ] 下单页券默认勾选且可取消；取消后支付券仍可用  
- [ ] Day28 三种分支文案与主按钮商品方向正确
